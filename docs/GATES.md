# Gates

Execution in this repo is gated. A gate is a discrete unit of work with a clear definition of done. Builder (Claude Code) does not advance to the next gate until the current one is signed off by the Gatekeeper (ChatGPT) with the Founder's approval.

---

## GATE: ZIDNI-ACADEMY-REPO-0

**Purpose:** initialize repo, SSOT, Hermes onboarding, and first project structure.

**Definition of done:**

- Required files exist:
  - `README.md`
  - `docs/SSOT.md`
  - `docs/FOUNDER_CONTEXT.md`
  - `docs/HERMES_ONBOARDING.md`
  - `docs/COURSE_BLUEPRINT.md`
  - `docs/PUBLIC_MOVEMENT_STRATEGY.md`
  - `docs/FACEBOOK_GROWTH_PLAN.md`
  - `docs/GATES.md`
  - `evidence/README.md`
  - `hermes/learning_log.md`
  - `hermes/founder_profile.md`
  - `hermes/project_memory.md`
- Brand name is consistent (**Zidni Academy** everywhere; no other variant).
- Hermes onboarding exists.
- Course blueprint exists.
- Facebook growth plan exists.
- Commit pushed to `main`.
